GENOME · Memory Layer Benchmark · v1.0

An honest read of GENOME on LoCoMo & TempBelief

GENOME ingests with zero LLM calls (over 8,000× cheaper at Haiku pricing), beats truncated full-context once history overflows the window, and keeps a deterministic, auditable bi-temporal record. It answers point-in-time queries better than Mem0 when sources state explicit dates (0.870 vs 0.676) — though we measured that edge disappearing on natural relative-dated language, and we say so. On saturated in-window QA it ties Mem0. Where its features don't help, this report says that too.

METHODThe comparison is fair by construction

Every system in this report answers with the same model, is graded by the same judge (the Mem0-verbatim judge prompt at temperature 0), embeds with the same encoder (text-embedding-3-small), and retrieves the same top-k. The only thing that varies is the memory layer. That isolates the layer as the independent variable and removes the usual confound where a "better" score just reflects a bigger answer model or a laxer judge.

Scores are reported over the 1,540 headline questions (LoCoMo categories 1–4: multi-hop, temporal, open-domain, single-hop). The adversarial category 5 is scored separately as an abstention test. Significance is paired McNemar on per-question correctness — the right test when the same questions are answered by both systems.

Two caveats we state up front. (1) The answer model and the judge are the same family (Haiku 4.5) — standard for LoCoMo harnesses, but a same-model-judge carries a self-preference risk that applies equally to every system here. (2) Numbers are single-run at a fixed seed; we report point estimates without confidence intervals, so treat sub-percent gaps as noise.

FINDING 06 · WIN (EXPLICIT-DATED SOURCES ONLY)Point-in-time truth — but only when dates are explicit

The findings above are about retrieval. This one is a different primitive. GENOME maintains a contradiction-resolved, bi-temporal model of how facts change, recording each fact at its domain-time validity (when it became true in the world, parsed from the text) rather than wall-clock ingest time. So it can answer “what was true at time T?” even when facts are revealed out of chronological order. We tested this on TempBelief -- a deterministic, gold-validated benchmark where entities' attributes change over 6-8 sessions, ~1/3 of updates arrive out of order, and distractors must not move the belief. Same responder/judge/embedder; Mem0 run at its best (infer=True + history() traversal).

The honest read: on point-in-time queries, belief-state beats every baseline with significance (vs Mem0 p=0.0007, vs dense and full-context p<10−5). Overwrite-based memory loses old values; raw retrieval and full-context return the latest value or can't disambiguate dates in a cluttered context. Only a bi-temporal store answers “what was true at T” reliably. This is not judge leniency: a full six-conversation audit finds the KG at 0.97 precision / 0.96 recall, and every captured fact is dated correctly (domain-time accuracy 1.000). Honest tradeoff: belief-state loses current-value (0.833 vs 0.97-1.0) -- its LLM extraction is noisier on trivial “latest value” lookups than reading the last mention.

The win does NOT survive natural language — we measured it. We re-ran the benchmark with every fact phrased relatively (“about two years ago”, “last year”) anchored to the message date, instead of a verbatim “Month Year”. The result: as-of belief 0.759 vs Mem0 0.722, McNemar p=0.82 — a tie. The decisive verbatim result (0.870 vs 0.676) collapsed, because with relative dates Mem0's LLM resolves them at query time about as well as GENOME resolves them at ingest. So this is a real advantage only when the source states explicit, machine-parseable dates (logs, structured notes, timestamped records) — NOT a general-conversation advantage. The durable belief-state value is the deterministic, structured, auditable record (below), not a QA-accuracy edge. Also: the history row is a partial-credit ceiling (all four at 1.000, uninformative); the real signal is as-of alone, not macro-J; 6 convs, one synthetic dataset, single seed.

Paired McNemar finds no significant difference between GENOME and either baseline: versus Mem0, Δ=−0.004 (p=0.747); versus full-context, Δ=−0.012 (p=0.232). Both gaps are within noise — but note the direction: GENOME's point estimate is nominally the lowest of the three.

The honest read: LoCoMo in-window accuracy has a ceiling near 0.85 that every serious system hits. GENOME reaches it but sits a hair below both baselines; the differences aren't statistically significant, so the correct statement is “no confirmed difference,” not “GENOME ahead.” Either way, this is not where a memory layer earns its keep — and we don't claim an in-window accuracy win.

The honest read: the robust, model-independent fact is the call count — 0 vs 1.00 LLM call per message, which no pricing choice can change. The dollar multiplier does depend on the model priced: at Haiku pricing the full corpus is ~$52 (Mem0) vs ~$0.006 (GENOME); a cheaper extraction model shrinks the ratio but it stays large, because GENOME's ingest LLM cost is structurally zero. One mechanism we did not measure but expect: Mem0's per-message cost should rise as its store fills (each write ships existing memories to the LLM to decide an update), so linear extrapolation is if anything an underestimate. GENOME's differentiator is the zero in that first column.

FINDING 04 · HONEST NULLAmong modes we could fairly test, dense wins

The honest read: among the modes this harness could fairly exercise, plain dense is best — hybrid is worse (0.564 vs 0.715 hit@10, a descriptive mean over 1,536 questions; keyword blending pulls in lexically-similar but wrong turns) and recency reranking is slightly worse (LoCoMo evidence spans all sessions). Graph mode is omitted, not validated: it needs an entity graph this offline store never built, so it correctly fell back to dense and can't be scored here (we confirmed the store held 0 entities). We make no claim that graph helps — and an earlier full-pipeline graph variant scored far worse on answers (0.494), so the burden is on graph to prove itself. Dense is GENOME's validated default.

FINDING 05 · HONEST NULLSynthesis is a tradeoff, not a free win

GENOME's signature idea is consolidation: rather than forget old memories under a budget, summarize them. We tested it the only fair way — equal token budget on both sides. Earlier internal runs showed a large synthesis "win," but that was a confound: a cluster-summary is a dense fact-list fed far more context than a pruned turn. Held to equal tokens, the picture is honest.

Storage budget S=3,000 tok, answer budget B=1,500 tok, n=584 (all 4 convs). Context fed was equal (1,473 vs 1,436 tok). Delta +0.005, McNemar p=0.861 — not significant. Source: benchmarks/budget_fair.py.

The honest read: aggregate delta is +0.005 — statistically zero (p=0.86). The per-category pattern is directional, not individually significance-tested (only the aggregate was): synthesis trends up on cross-session reasoning (temporal +0.085, multi-hop +0.072) and down on exact-fact lookup (single-hop −0.048, open-domain −0.031), consistent with the mechanism — a summary blurs the one verbatim detail a single-hop question needs. Read as a tendency, not proof. Consolidation is a tunable tradeoff: turn it on for temporal/analytical workloads, off for lookup-heavy ones. It is not a blanket accuracy win, and we don't sell it as one.

GUIDANCEWhen GENOME is the right choice

Choose GENOME when ingestion cost matters (high message volume, or you don't want an LLM in your write path), or when conversation history exceeds the model's context window — the two cases this report measures a real advantage in.

Don't expect a higher in-window LoCoMo score than Mem0 — that metric is saturated and the two tie. If your entire history already fits the window and ingestion cost is irrelevant, full-context is a fine and simpler baseline; a memory layer only starts paying off past that point.
